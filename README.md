# HR Sentiment Analytics MVP

MVP-система для анализа эмоционального фона сотрудников на основе текстовых отзывов

Проект разработан в рамках курсовой работы.
Цель — проверить, можно ли использовать методы аализа текста и машинного обучения для HR-мониторинга: определения тональности отзывов, выявления негативных сигналов и основных HR-тем

## Что есть в проекте

* сбор отзывов сотрудников с DreamJob;
* очистка и подготовка текстов;
* автоматическая разметка отзывов через рейтинг;
* дополнительная проверка разметки с помощью языковой модели;
* baseline-модель анализа тональности: TF-IDF + Logistic Regression;
* выделение HR-тем;
* Streamlit-интерфейс для визуализации результатов.

## Основные файлы

* `parser_dreamjob.py` — сбор отзывов
* `ml_pipeline.py` — обучение модели и оценка качества
* `app.py` — Streamlit-интерфейс
* `make_labeling_sample.py` — создание выборки для проверки разметки
* `evaluate_labeling.py` — сравнение rating-разметки и LLM-разметки
* `requirements.txt` — зависимости проекта

## Данные и результаты

* `tbank_reviews.csv` — собранные отзывы
* `tbank_reviews_with_predictions.csv` — отзывы с предсказаниями модели
* `sample_for_labeling_labeled.xlsx` — выборка с дополнительной разметкой
* `labeling_comparison_report.csv` — отчёт по проверке разметки
* `sentiment_model.pkl` — сохранённая модель

## Запуск

Установка зависимостей:

```bash
pip install -r requirements.txt
```

Запуск обучения модели:

```bash
python ml_pipeline.py
```

Запуск интерфейса:

```bash
streamlit run app.py
```

## Результаты

Baseline-модель показала:

* accuracy: около 0.83
* F1 для positive-класса: около 0.90
* F1 для negative-класса: около 0.56

Дополнительная проверка разметки на 100 отзывах показала:

* совпадение rating-разметки и ручной разметки: 74%
* потенциально шумные случаи: 26%

## Ограничения

Проект является исследовательским MVP, а не готовым промышленным решением
Основные ограничения: шумная разметка через рейтинг, низкое качество определения негативного класса и использование простой baseline-модели
